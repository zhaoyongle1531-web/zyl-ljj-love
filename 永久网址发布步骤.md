# 永久网址发布步骤

推荐路线：

GitHub 私有仓库 -> Cloudflare Pages 自动部署 -> 永久网址。

这样以后你更新照片或清单，只需要提交到 GitHub，Cloudflare 会自动更新网站。

## 第一次发布

### 1. 创建 GitHub 私有仓库

1. 打开 https://github.com/new
2. Repository name 填：
   `zyl-ljj-love`
3. Visibility 选择：
   `Private`
4. 不要勾选初始化 README、.gitignore、license。
5. 点击 `Create repository`。

### 2. 推送本地项目

GitHub 创建完成后，会给你一个仓库地址，类似：

`https://github.com/你的用户名/zyl-ljj-love.git`

在项目目录运行：

```powershell
git remote add origin https://github.com/你的用户名/zyl-ljj-love.git
git branch -M main
git push -u origin main
```

### 3. 连接 Cloudflare Pages

1. 打开 https://dash.cloudflare.com/
2. 进入 `Workers & Pages`
3. 点击 `Create application`
4. 选择 `Pages`
5. 选择 `Connect to Git`
6. 授权并选择 GitHub 仓库：
   `zyl-ljj-love`
7. 构建设置填写：
   - Framework preset: `None`
   - Build command: 留空
   - Build output directory: `publish_site`
8. 点击 Deploy。

发布完成后，Cloudflare 会给你一个长期网址，通常类似：

`https://zyl-ljj-love.pages.dev`

## 日常更新

### 更新一起做的事

编辑：

`content.json`

找到 `moments`，新增或修改文字。

### 更新照片

1. 把新照片放进：
   `assets/original_photos`
2. 双击：
   `更新网站.bat`
3. 双击：
   `打开爱情网页.bat`
4. 确认本地效果没问题后运行：

```powershell
git add .
git commit -m "Update love website"
git push
```

Cloudflare Pages 会自动部署，永久网址保持不变。

## 隐私建议

- GitHub 仓库建议设为 Private。
- `.gitignore` 已忽略原始大图，只把压缩后的网页图片发布出去。
- 只要拿到 Pages 网址的人就能访问网页，不要把链接发到公开平台。
