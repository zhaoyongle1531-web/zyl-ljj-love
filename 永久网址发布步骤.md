# 永久网址发布步骤

你现在看到的是 Cloudflare 新版 `Create a Worker` 页面。这个页面可以发布静态网站，但第一次必须先注册 `workers.dev` 子域名，否则会出现：

`You need to register a workers.dev subdomain before publishing to workers.dev`

## 解决办法

### 1. 注册 workers.dev 子域名

打开：

https://dash.cloudflare.com/?to=/:account/workers/onboarding

注册一个账号级别的子域名，例如：

`zyl-ljj`

这是一次性设置。注册后，你的网站永久网址通常会类似：

`https://zyl-ljj-love.zyl-ljj.workers.dev`

### 2. 重新部署

回到你截图里的页面：

1. 选择 `Continue with GitHub`
2. 选择仓库 `zyl-ljj-love`
3. 点击 Deploy

项目根目录已经包含：

`wrangler.jsonc`

它会告诉 Cloudflare：

- 网站名称：`zyl-ljj-love`
- 发布目录：`publish_site`
- 静态资源绑定：`ASSETS`

### 3. 如果还是失败

进入失败项目的 Settings，确认部署命令不是手动写死的错误命令。

可以保留默认部署，也可以使用：

```bash
wrangler deploy
```

不要使用旧的 Pages 配置项，因为你当前账号界面走的是 Worker Static Assets。

## 日常更新

### 更新一起做的事

编辑：

`content.json`

找到 `moments`，新增或修改文字。

### 更新照片

1. 把新照片放进：
   `assets/original_photos`
2. 双击：
   `更新网站.bat`
3. 双击：
   `打开爱情网页.bat`
4. 确认本地效果没问题后运行：

```powershell
git add .
git commit -m "Update love website"
git push
```

Cloudflare 会自动部署，永久网址保持不变。

## 隐私建议

- GitHub 仓库建议设为 Private。
- `.gitignore` 已忽略原始大图，只把压缩后的网页图片发布出去。
- 拿到永久网址的人就能访问网页，不要把链接发到公开平台。
